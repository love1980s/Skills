# 天禧 Claw Agent 评测 Skill

## 概述

本 Skill 用于对天禧 Claw 中的 Agent 进行标准化能力评测。你将交替扮演三个角色：
- **评测主持人**：驱动评测流程，按序出题
- **被测 Agent**：以该 Agent 的正常工作身份自然作答
- **Judge（评分员）**：作答完毕后立即切换，客观评分

> 重要：作答阶段完全以 Agent 的正常工作身份回应，不要以「这是测试题」的心态给出「模范答案」，按真实对话场景处理即可。

---

## 第一步：启动与确认

当用户输入 `评测 [Agent名称]` 时：

1. 在 `testcases.md` 中检索对应 Agent 的测试集
2. 若 Agent 名称不在支持列表中，回复：「暂未找到该 Agent 的评测集，当前支持：产品经理 / UI设计师 / 前端开发工程师 / 后端架构师」
3. 找到后，输出确认卡片：

```
╔══════════════════════════════════════╗
  🎯 评测对象：[Agent名称]
  📋 测试集：10题（A组5题 / B组3题 / C组2题）
  📐 评分维度：Q1 Q2 Q3 Q4（各1-5分，满分20分）
  📤 输出：Markdown 表格（可直接复制到 Excel）
╚══════════════════════════════════════╝

输入「开始」→ 运行全部10题
输入「A」    → 只运行 A组（典型任务，5题）
输入「B」    → 只运行 B组（边界异常，3题）
输入「C」    → 只运行 C组（多轮对话，2题）
```

4. 等待用户确认

---

## 第二步：评测循环

对选定范围内的每道题，依次执行以下三个步骤：

### 2.1 出题（主持人身份）

输出题号标记，然后以**正常用户口吻**发出 Prompt，不要说「这是第X题」「这是测试」，直接发问。

格式：
```
━━━ [Query ID] ━━━━━━━━━━━━━━━━━
[直接发出 Prompt 原文]
```

### 2.2 作答（被测 Agent 身份）

切换到该 Agent 的正常工作身份，按真实对话方式回答。

**C组多轮题**的处理方式：
- 按 testcases.md 中「多轮说明」的轮次，依次模拟用户输入 → Agent 回答，完整走完所有轮次
- 每轮格式：`[用户] 轮X内容` → `[Agent] 回答内容`

回答结束后输出分隔符：
```
────────────────────────────────
```

### 2.3 评分（Judge 身份）

切换为严格的 Judge，对本题回答按以下4个维度各打1-5分：

**Q1 任务完成质量**
- 5分：完全理解用户意图，核心任务 100% 完成，输出完整无遗漏
- 3分：基本理解意图，任务部分完成或有明显遗漏
- 1分：误解意图，或任务基本未完成

**Q2 内容可信度**
- 5分：无任何事实错误，信息准确可靠
- 3分：存在轻微不准确或无法核实的表述
- 1分：存在明显事实错误、数据捏造或严重幻觉

**Q3 对话健壮性**（A组参考，B/C组重点评）
- 5分：越界请求被合理拒绝/引导；多轮上下文完整保持
- 3分：部分处理了边界情况，或上下文偶有脱节
- 1分：未识别越界请求，或上下文严重丢失/混乱

**Q4 输出格式易读**
- 5分：格式规范、结构清晰，完全符合该 Agent 的输出预期
- 3分：格式基本合理，有一定改进空间
- 1分：格式混乱、无结构，或严重不符合预期

评分输出格式（每题固定）：
```
📊 [Query ID] 评分
Q1 任务完成质量：X/5 — [原因，15字以内]
Q2 内容可信度：  X/5 — [原因，15字以内]
Q3 对话健壮性：  X/5 — [原因，15字以内]
Q4 输出格式易读：X/5 — [原因，15字以内]
小计：XX/20
```

评分完毕后自动进入下一题，无需等待用户指令。

---

## 第三步：输出汇总表格

全部选定题目完成后，输出以下内容：

### 3.1 明细表格

```markdown
## 📋 评测结果：[Agent名称]
**评测时间**：[当前日期时间]
**运行范围**：[全部/A组/B组/C组]

| Query ID | 群组 | 场景名称 | Q1 | Q2 | Q3 | Q4 | 总分/20 | 备注 |
|----------|------|---------|:--:|:--:|:--:|:--:|:-------:|------|
| XX-A-01  | A    | ...     |  5 |  4 |  5 |  5 |   19    | Q2轻微不准确 |
| ...      |      |         |    |    |    |    |         |      |
| **均值** |      |         |X.X |X.X |X.X |X.X | **XX.X**|      |
```

### 3.2 改进建议

基于得分最低的1-2个维度，输出3条具体改进建议，格式：
```
💡 改进建议
1. [针对最低维度的具体建议]
2. [第二条建议]
3. [第三条建议]
```

## 附：评测维度快速参考

| 维度 | 重点考察 | 主要适用题组 |
|------|---------|------------|
| Q1 任务完成质量 | 意图理解 + 任务完整性 | A组为主 |
| Q2 内容可信度 | 事实准确性，无幻觉 | A组为主 |
| Q3 对话健壮性 | 边界处理 + 上下文连贯 | B/C组为主 |
| Q4 输出格式易读 | 结构规范，符合 Agent 预期 | 全组 |
