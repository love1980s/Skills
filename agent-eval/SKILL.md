# 天禧 Claw Agent 评测 Skill

## 概述

本 Skill 分两个阶段运行：

- **Phase 1 出题与作答**：以被测 Agent 身份自然作答所有题目，不评分、不提示预期
- **Phase 2 生成 Excel**：将所有 Q+A 写入 Sheet1，将评分标准写入 Sheet2，生成本地文件供第三方模型独立评分

---

## 第一步：启动与确认

当用户输入 `评测 [Agent名称]` 时：

1. 读取 `testcases.md`，检索对应 Agent 的测试集
2. 若名称不在支持列表中，回复：「暂未找到该 Agent 的评测集，当前支持：产品经理 / UI设计师 / 前端开发工程师 / 后端架构师」
3. 找到后输出确认卡片：

```
╔══════════════════════════════════════╗
  🎯 评测对象：[Agent名称]
  📋 测试集：10题（A组5题 / B组3题 / C组2题）
  📤 输出：本地 Excel（答卷 + 评分标准，供第三方独立评分）
╚══════════════════════════════════════╝

输入「开始」→ 运行全部10题
输入「A」   → 只运行 A组（典型任务，5题）
输入「B」   → 只运行 B组（边界异常，3题）
输入「C」   → 只运行 C组（多轮对话，2题）
```

4. 等待用户确认

---

## 第二步：Phase 1 — 出题与作答

对选定范围内每道题，依次执行：

### 2.1 出题

输出题号标记，然后以**正常用户口吻**发出问题，不说「这是测试」，直接发问：

```
━━━ [Query ID] ━━━━━━━━━━━━━━━━━
[题目原文]
```

若题目有附件引用（`📎 上传文件`），说明文件内容已提供，直接基于该文件内容作答。

### 2.2 作答

以该 Agent 的**正常工作身份**回答，按真实对话场景处理，不受评测背景影响。

**C组多轮题**：按「多轮说明」的轮次依次模拟，每轮格式：
```
【用户 轮X】[用户输入]
【Agent 轮X】[回答]
```

作答结束后输出分隔符：
```
────────────────────────────────
```

**所有题目作答完毕后，不做任何评分，直接进入第三步。**

---

## 第三步：Phase 2 — 生成 Excel

所有 Q+A 完成后，使用 bash + openpyxl 生成本地 Excel 文件。

### 3.1 文件路径

保存至工作区文件夹：`[workspace]/[Agent名称]_评测答卷_[YYYYMMDD].xlsx`

### 3.2 Sheet1 "答卷" 结构

| 列 | 内容 |
|----|------|
| A | Query ID（如 PM-A-01） |
| B | 分组（A / B / C） |
| C | 场景名称（如「PRD写作」） |
| D | 题目（完整题目文字） |
| E | 辅助文件（若有，填写文件名） |
| F | Agent 答案（完整作答内容） |
| G | Q1 得分（留空，供评测方填写） |
| H | Q2 得分（留空） |
| I | Q3 得分（留空） |
| J | Q4 得分（留空） |
| K | 总分（留空） |
| L | 评测备注（留空） |

### 3.3 Sheet2 "评分标准" 结构

Sheet2 写入以下固定内容（硬编码，无需从外部加载）：

**Section 1 — 通用评分维度（4个维度，每个1-5分）**

| 维度 | 考察重点 | 5分标准 | 3分标准 | 1分标准 |
|------|---------|---------|---------|---------|
| Q1 任务完成质量 | 意图理解 + 任务完整性 | 完全理解意图，核心任务100%完成，输出完整无遗漏 | 基本理解意图，任务部分完成或有明显遗漏 | 误解意图，或任务基本未完成 |
| Q2 内容可信度 | 事实准确性，无幻觉 | 无任何事实错误，信息准确可靠 | 存在轻微不准确或无法核实的表述 | 存在明显事实错误、数据捏造或严重幻觉 |
| Q3 对话健壮性 | 越界处理 + 上下文连贯 | 越界请求被合理拒绝/引导；多轮上下文完整保持 | 部分处理了边界情况，或上下文偶有脱节 | 未识别越界请求，或上下文严重丢失 |
| Q4 输出格式易读 | 结构规范，符合 Agent 预期 | 格式规范、结构清晰，完全符合该 Agent 的输出预期 | 格式基本合理，有一定改进空间 | 格式混乱、无结构，或严重不符合预期 |

**Section 2 — B组边界题参考行为**

为评测方提供各 B 组题目的参考判断依据：

| Query ID | 场景 | 参考行为描述 |
|----------|------|-------------|
| PM-B-01 | 越界技术请求 | 应识别代码开发超出 PM 职责范围，建议咨询技术 Agent，不应直接提供 Python 脚本 |
| PM-B-02 | 极度模糊需求 | 应主动追问，至少澄清目标用户、使用场景、核心价值三个维度，不应直接给出功能方案 |
| PM-B-03 | 不合理信息请求 | 应说明竞品内部后台数据无法获取，提供可替代的公开数据获取方案（如第三方数据平台、App Store 排名等） |
| UI-B-01 | 超出能力范围 | 应说明无法直接生成 Figma 文件，但应提供完整的文字规范或设计说明作为替代 |
| UI-B-02 | 需求极度模糊 | 应主动追问业务场景、目标
