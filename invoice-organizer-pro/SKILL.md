# 差旅发票整理与比对专家 (Pro)

## 核心功能

1. **自动识别与重命名**：读取 PDF 内容，识别类型（酒店/打车/餐饮），按标准格式重命名。
2. **报销明细比对**：读取 Excel 报销单，提取每一项的“申请金额”和“日期”。
3. **合规性验证**：确保发票金额覆盖申请金额，检查日期偏差，提示可能的遗漏项。

## 命名规则

格式：`金额-类型-详情-[人数]-日期-文件性质.pdf`

| 字段 | 说明 | 示例 |
|------|------|------|
| 金额 | 实际消费金额，保留两位小数 | `109.20` |
| 类型 | `酒店` / `打车` / `餐饮` | `打车` |
| 详情 | 地点或路线，15字以内 | `北京首都机场到联想总部` |
| 人数 | **仅打车填写**，格式 `X人` | `1人` |
| 日期 | `MMDD` 或范围 `MMDD至MMDD` | `0309` |
| 文件性质 | `发票` / `结账单` / `行程单` | `发票` |

## 工作流程

### 第一步：扫描与识别
使用 `scripts/extract_pdf.py` 提取 PDF 文本。
- **酒店**：识别“住宿服务”、“入离日期”。
- **打车**：识别“运输服务”、“行程起止日期”、“金额”。
- **餐饮**：识别“餐饮服务”、“商户名称”。

### 第二步：读取报销表格
使用 `scripts/compare_excel.py` 读取目录下的 Excel 文件（通常包含“报销”或“出差”字样）。
- 提取关键列：`日期`、`费用类型`、`金额`、`已申请金额`。

### 第三步：比对分析
将识别出的发票数据与 Excel 每一行进行匹配。
- **金额校验**：发票金额必须 >= 表格中的申请金额。
- **日期校验**：发票日期应在出差周期内或与之匹配。
- **配对校验**：确保打车发票有对应行程单，酒店发票有对应结账单。

### 第四步：展示方案并执行
1. 展示拟重命名清单。
2. 列出比对结果（差异、遗漏、风险点）。
3. 等待用户确认后，使用 `mv` 执行重命名。

## 脚本使用参考
```bash
# 提取 PDF 文本
python scripts/extract_pdf.py "path/to/invoice.pdf"

# 读取 Excel 明细
python scripts/compare_excel.py "path/to/reimbursement.xlsx"
```

## 注意事项
- **权限问题**：如果 Excel 被占用，脚本会自动尝试复制副本读取。
- **日期敏感**：优先使用消费日期而非开票日期。
- **金额精度**：始终保留两位小数。
